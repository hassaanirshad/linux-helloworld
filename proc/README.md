## What it does

Demonstrates different ways to create child processes and threads on Linux. Each command spawns one or more children, performs a file operation (open/write/read a temp file) in both the parent and each child, and logs PID/TID/PGID/PPID info to stderr.

| Command | Mechanism | Notes |
|---|---|---|
| `fork` | `fork(2)` | Classic process fork |
| `vfork` | `vfork(2)` | Parent suspends until child `_exit`s |
| `clone_non_thread` | `clone(2)` (SIGCHLD) | New process, separate address space |
| `clone_single_thread` | `clone(2)` (CLONE_THREAD \| ...) | New thread sharing address space |
| `clone_multi_thread` | `clone(2)` × 2 | Two threads sharing address space |
| `clone_non_thread_execve` | `clone(2)` + `execve(2)` | Child replaces image via execve |
| `clone_single_thread_execve` | `clone(2)` + `execve(2)` | Thread replaces entire process image |
| `clone_multi_thread_execve` | `clone(2)` × 2 + `execve(2)` | Two threads, then parent execs |

## Build

```sh
cd proc
make
```

Binary is placed at `bin/main_proc`.

## Run

```sh
bin/main_proc --command <command> [--verbose]
```

`--verbose` enables detailed file operation logging (open/write/read steps).

### Examples

```sh
bin/main_proc --command fork
bin/main_proc --command clone_multi_thread --verbose
bin/main_proc --command clone_non_thread_execve
```

## Clean

```sh
make clean
```
