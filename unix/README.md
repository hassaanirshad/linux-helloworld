## What it does

A Unix domain socket server/client pair that demonstrates passing open file descriptors between processes using `SCM_RIGHTS` ancillary data via `sendmsg`/`recvmsg`.

**Flow:**
1. Server starts, binds a Unix socket, and waits for one client connection.
2. Client connects and invokes the command handler.
3. For `fd_transfer`:
   - Server creates a temp file (`tmpfile`), writes `"hello from server"` into it, and sends the open fd to the client over the socket.
   - Client receives the fd, prints its info (`fstat` + `/proc/self/fd`), writes `"hello from client"` into it, then sends `"SUCCESS"` or `"FAILURE"` back.
   - Server receives the response and exits.

## Build

```sh
cd unix
make
```

Binaries are placed in `bin/`:
- `bin/main_server`
- `bin/main_client`

## Run

Start the server in one terminal:

```sh
bin/main_server --unix-sock-path /tmp/test.sock --command fd_transfer
```

Connect with the client in another:

```sh
bin/main_client --unix-sock-path /tmp/test.sock --command fd_transfer
```
